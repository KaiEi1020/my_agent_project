# 三个工程难点

1. 工具双向验证（Tool Call Verification）： 如何捕获大模型的工具调用意图，对它的参数进行高精度的业务级合法性检查（Sanitization）。

2. 动态挂起与中断（Graph Interruption / Breakpoints）： 开启 LangGraph 的 interrupt_before 机制。当大模型试图调用“转账”、“删除”等高危工具时，整个图状态会瞬间在这一步被冻结（挂起），等待人类经理的指令。

3. 人机反馈注入（Human Feedback Injection）： 人类经理在前端后台点击“同意”或“拒绝，并修改转账金额”。Agent 会把人类的意志无缝灌入 State，解冻状态机，继续向下流转。

## 断点

```python
# 配置的是节点名称列表, 每次图执行到即将进入 execute_bank_transfer 节点之前，强制暂停。
app = workflow.compile(
    checkpointer=memory,
    interrupt_before=["execute_bank_transfer"] # 在进发网银前，时空拉闸！
)
```



**发现金额错误！执行方案 B：人类直接强行干预，重写 State 里的金额**

```python

app.update_state(
    config_thread,
    {"transfer_amount": 1000}, # 覆写状态！

)
```





**checkpoint 底层状态**（本质是链表结构）

```python
{
    "channel_values": {
        "transfer_amount": 100000,
        "messages": [...],
    },
    "next": ["execute_bank_transfer"],  # 下一步执行谁
    "source": "auditor_brain",          # 这条记录是谁写的  ← 这就是 as_node 控制的
}
```



**继续执行, 直接从断点位置往后跑**

传入非 `None` 的初始状态会被解释为**新的任务触发**，图会重置从头执行，`interrupt_before` 就会再次生效。

```python
final_state = app.invoke(None, config=config_thread)
```
