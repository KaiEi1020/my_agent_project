# 记忆视界管理（Memory Horizon Management）

#### 手段一：硬性边界裁剪（Trimming / Slidewindow）

这是最廉价、最高效的手段。利用 `langchain_core.messages` 自带的 `trim_messages` 工具，在每次把消息列表喂给 `model.invoke()` 之前，用 Python 脚本在内存中直接执行==“滑动窗口”==裁剪。

例如：永远只保留最开始的 `SystemMessage`（身份和红线）以及最近的 4 条消息。中间那些“初审算错”、“人类经理的第一轮痛骂”等已经过时的交锋历史，直接从当前的输入 Payload 中抹除。



#### 手段二：滚动语义总结（Incremental Summarization）

当图流转到特定节点，或者消息数触及阈值时，触发一个后台的 `summarize_node`。 这个节点专门调用一次低成本的 Flash 模型，把过往的十几条对话压缩成一句话：“*前情提要：用户要求审计 Q1 财报，中途因总营收与利润混淆被人类经理驳回，现已明确利润为 1000 元。*” 然后，系统将这个 Summary 替换掉之前的全部历史，作为唯一的历史背景喂给大模型。
